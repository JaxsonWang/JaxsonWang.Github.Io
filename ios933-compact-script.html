iOS9.3.3一键精简系统脚本

发表于 2016-08-03 | 更新于 2018-09-29 | 分类于 奇淫技巧

简介

这是一份适合iOS9.2-9.3.3越狱后一键精简系统的脚本，可达到对ROM，RAM进行一些列的优化。用户在脚本选择可以进行交互，某些应用程序是否保留有用户说了算。且精简的应用程序会自动备份在用户内存中，用户可以自己手动恢复应用程序。

此脚本不适合小白，遇到一些问题请手动解决，若发现比较严重的问题请联系我！

拜托各位推广了，感谢！

运行准备

在运行脚本之前请关闭设置-通用-访问限制选项统统关闭！
在运行脚本之前请关闭设置-隐私-定位服务！！
否则会出现白图标！

然后使用ftp上传脚本，把black文件夹放在~目录，也就是/private/var/root目录`，打开SSH终端输入命令：

cd ~ && cd black && chmod 644 black.sh && sh black.sh

若是手机运行脚本，需要root登录：

login
cd ~ && cd black && chmod 644 black.sh && sh black.sh

剩下按照脚本的提示来运行！

关于恢复App一些思路引导

最近时间有点紧张，脚本好多地方要完善。实在是挤出时间在修复脚本一些存在的bug。
关于恢复的脚本，不出意外这个星期搞定。有些同学等不住可以自己动手恢复，给大家一点思路。

从备份的App直接拉到根目录，这点没错，重启后桌面确实出现App图标了，但打开出现闪退，那是因为没设置权限。

而我的脚本打权限是：

chown -R mobile.mobile /private/var/mobile/Media/Backup/
chmod 755 /private/var/mobile/Media/Backup/
find /private/var/mobile/Media/Backup/ -type d -exec chmod 755 {} ;
find /private/var/mobile/Media/Backup/ -type f -exec chmod 644 {} ;

考虑到读写权限问题，我默认设置用户群组为mobile，但在system可是root组。

从终端得知结果
so，需要再加上几条条命令:

chown -R root.admin /YourApplications
find /YourApplications -type d -exec chmod 755 {} ;
find /YourApplications -type f -exec chmod 644 {} ;

/YourApplications就是你所需要恢复的文件目录
建议大家在恢复的时候看看其他文件权限用户组是什么，那就改什么。
